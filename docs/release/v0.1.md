# Forge v0.1 — MVP Release

**Released:** 2026-04-14

Forge v0.1 is the minimum viable product spanning Layers 1–4: a typed blueprint engine, an intelligence harness, a factory pipeline for isolated runs and delivery, and integration polish with CI and documentation.

---

## Layer 1: Blueprint Engine (Go)

- **Typed graph engine** with three node types:
  - **AgenticNode** — LLM agent steps
  - **DeterministicNode** — shell commands with timeout
  - **GateNode** — conditional pass/fail routing
- **YAML blueprint parser** — two-phase parse and build
- **RunState** — tracking with per-node results
- **Iteration guard** on gate loops
- **`AgentExecutor` interface** — pluggable agents
- **Built-in blueprints:** `standard-implementation`, `bug-fix`
- **CLI:** `forge blueprint validate|list|run`
- **Extras:** parallel node execution (`engine_parallel.go`), lifecycle hooks (`hooks.go`), permission model (`permissions.go`)
- **Quality:** 75 tests, 91% coverage

---

## Layer 2: Intelligence Harness (TypeScript)

- **gRPC ForgeAgent service** — `proto/forge/v1/agent.proto`
- **Agent adapters:** Echo (testing), Claude Code (headless CLI)
- **Context loader** — `AGENTS.md` plus `.forge/rules/` scanning
- **Go gRPC client** — `GrpcAgentExecutor`
- **Quality:** 21 TypeScript tests (100% statement coverage), 4 Go gRPC tests (94.1%)
- **Full e2e integration test** — Go engine with TypeScript harness

---

## Layer 3: Factory MVP (Go)

- **Docker sandbox** — container lifecycle, image management, resource limits, network policy
- **Git worktree isolation** for parallel runs
- **Delivery pipeline** — git push and PR creation via `gh` CLI
- **Pipeline orchestrator** — workspace → sandbox → delivery
- **Dockerfile** and sandbox entry script
- **CLI:** `forge run "task"`
- **Quality:** 35+ tests, 100% statement coverage

---

## Layer 4: Integration and Polish

- Blueprint source resolution and task templating
- Local mode alignment
- **CI** — GitHub Actions (Go, TypeScript, Docker)
- README quickstart
- Deterministic smoke tests
